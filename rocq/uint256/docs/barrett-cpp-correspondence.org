* Purpose

This note supports manual side-by-side review of
=rocq/uint256/theories/Barrett.v= against the Barrett implementation in
=category/core/runtime/uint256.hpp=.

The correspondence is kept in a separate document rather than embedded as
comments on every Rocq definition.  The Barrett model is proof source, so large
line-by-line C++ comments would make the proof file harder to edit.  This
document can also record cross-cutting caveats, such as total zero-padded word
access, without overloading individual definitions.

The main non-literal detail here is the final-word branch of =rshift=.  The C++
chooses between =shrd(high, low, bit_shift)= and =low >> bit_shift= depending
on whether the high input word is in bounds.  The current Rocq model uses total
zero-padded =get_word= and therefore represents the out-of-bounds case as
=shrd64 zero low bit_shift=.  This is documented in
[[file:barrett-modeling-deviations.org::*=rshift= final-word branch is compressed by total zero-padded word access][barrett-modeling-deviations.org]].

* In-Progress C++ Review Notes

These notes record review clarifications during a manual pass-through.

- The reciprocal constructor proof comments are best read as informal
  instances of =barrett_floor_no_preshift_bound=.  The no-multiplier
  constructor uses scale =k = 1=; the constant-multiplier constructor uses
  =k = y=.
- In those comments, the proof inequalities use =x/d= and =xy/d= as exact
  mathematical quotients, but the algorithmic variable =q= should denote the
  integer floor quotient: =floor(x/d)= or =floor(x*y/d)=.
- The final form =q_hat <= q < q_hat + 2= is equivalent over integers to the
  no-pre-shift theorem statement =q - 1 <= q_hat <= q=.  It should not be
  confused with the pre-product-shift case, which has the weaker bound
  =q - 2 <= q_hat <= q= and is handled by =estimate_q= / =PRE_PRODUCT_SHIFT=.
- The Rocq proof =barrett_floor_no_preshift_bound= follows the same argument
  as the C++ sketch, but it encodes floor facts using integer-division
  inequalities such as =d * r <= k * 2^S= and
  =k * 2^S < d * (r + 1)= instead of rational subtraction forms.
- If the dormant =BIT_SHIFT != 0= multiplier numerator branch is retained, it
  needs the final carry word after the =shld= loop.  This matches
  =Div.shift_left_words=, which produces one more word than its input.

* Review Checklist

Use this list when comparing the files side by side:

- Check that derived constants in =Barrett.v= match the corresponding
  =static constexpr= formulas.
- Check that =numerator= and =numerator_with_multiplier= match the reachable
  C++ constructor behavior.
- Check =rshift= using the deviation note for the final-word branch.
- Check that =estimate_q= preserves the two-shift/product structure exactly.
- Check that =reduce= preserves the correction ordering and quotient updates.
- Check wrapper inputs: 256-bit =udivrem=, 257-bit =addmod=, 512-bit
  =mulmod=, and 256-bit online input for =mulmod_const=.
