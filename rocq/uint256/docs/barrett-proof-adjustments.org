#+TITLE: Barrett Proof Adjustments Log
#+DATE: 2026-04-30

* Purpose

This note records theorem-strengthening and model changes made while proving
=rocq/uint256/theories/BarrettProofs.v=.  The rule for this phase is that
proof statements may be strengthened only when the added assumptions correspond
to C++ semantics or to documented semantic preconditions, not merely to make a
proof easier.

* Theorem-strengthening log

** Add admissible =BarrettParams= assumptions to constructor and wrapper theorems

The C++ type =barrett::reciprocal<Params>= has a template =requires= clause:

- =uint256_t{Params.min_denominator} > 0=
- =uint256_t{Params.min_denominator} <= uint256_t{Params.max_denominator}=
- =Params.input_bits > 0=

The Rocq model has a total =BarrettParams= record, so the corresponding
theorems must carry this admissibility as an explicit hypothesis.  Without it,
=constructor_sound_no_multiplier= is false.

Counterexample before strengthening:

- =min_denominator = 0=
- =max_denominator = 0=
- =input_bits = 1=
- =multiplier_bits = 0=
- =d = 0=

Then =valid_denominator params d = true=, because =valid_denominator= is only
the C++ runtime range check.  However, the constructed reciprocal stores a zero
denominator, while =reciprocal_invariant= requires =0 < denominator_Z rec=.

The proof file therefore introduces =params_admissible= as the semantic Rocq
form of the C++ template-level precondition and adds it to the constructor and
wrapper theorem statements that construct a reciprocal from =params= and =d=.

** Add multiplier storage-fit assumption to the multiplier constructor theorem

The C++ multiplier constructor stores =y= in a fixed
=words_t<MULTIPLIER_WORDS>= field and then asserts that the higher words of the
original =uint256_t= are zero.  The original Rocq theorem claimed
=multiplier_Z rec = to_Z_uint256 y= for every =y= whenever
=MULTIPLIER_WORDS > 0=, which is stronger than the C++ storage semantics.

Counterexample before strengthening:

- =multiplier_bits = 1=, so =MULTIPLIER_WORDS = 1=
- =y = 2^64=

The Rocq model, like the C++ object layout, stores only the low one-word
prefix of =y= in =multiplier_=.  The stored multiplier is zero, while
=to_Z_uint256 y = 2^64=.  The theorem therefore needs the semantic version of
the C++ debug assertion: =y= must fit in =MULTIPLIER_WORDS= words.

** Add proof-module dependencies to =BarrettProofs.MakeProofs=

The Barrett proofs rely on already-established facts about division, runtime
multiplication, and composite uint256 arithmetic.  The C++ semantics do not
change here; the proof functor is strengthened so it can use the existing
=DivisionProofs=, =RuntimeMulProofs=, and =ArithmeticProofs= results rather
than reproving those libraries locally.
